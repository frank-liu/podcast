# Tasks (Phase 2) — approach and top-level tasks

This file is an outline of task generation for the feature. Detailed tasks should be generated by the /tasks command using the templates.

Priority order (TDD, dependency-aware):

1. Research spikes (P0)
   - Prototype Google TTS synthesis for short summaries (PoC)
   - Validate hosting + RSS publish flow (PoC)
   - Legal: validate redistribution rights checklist

2. Tests (failing) — create tests first (P1)
   - Unit: summarizer behavior (truncate/prioritize to meet 600s budget)
   - Integration: aggregate messages -> summaries -> TTS mock -> audio concat -> metadata
   - Contract tests for publish endpoints (use OpenAPI)

3. Core implementation (P1)
   - Models: Publisher, Topic, Source, Message, Digest (DB migrations)
   - Services: Aggregator, Summarizer, TTS Adapter (Google), Publisher/Upload service
   - Worker: Job scheduler/worker (process digest jobs)

4. Publishing integration (P2)
   - Implement host upload adapter (S3 or host API)
   - Generate RSS entries or call host API

5. Frontend (optional, P2)
   - Topic management UI (React + shadcn/ui)
   - Publish logs view (digest history)

6. CI / QA (P1)
   - Add Constitution Check to pipeline: ensure spec has Constitution Check and Tone
   - Add contract and integration test jobs

7. Monitoring & Observability (P2)
   - Emit events/metrics for digest generation time, TTS latency, publish success/failures

Estimate: ~25–35 tasks; start with research spikes and failing tests.
